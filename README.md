# Лабораторна робота №4: IaC. Terraform. Ansible

## Опис проєкту
Цей репозиторій містить код для розгортання інфраструктури через Terraform та автоматизованого керування конфігурацією через Ansible. 
Застосунок розгортається на двох віртуальних машинах.

## Архітектура системи
Інфраструктура складається з 2-х ВМ на базі офіційного образу Ubuntu Jammy (провайдер VirtualBox):
1. **`worker-vm` (192.168.56.103)** — містить Nginx (Reverse Proxy) та веб-застосунок (Flask).
2. **`db-vm` (192.168.56.105)** — містить сервіс бази даних (MariaDB).

---

## Попередні вимоги
Перед запуском переконайтеся, що на вашому хості встановлено:
* **Terraform** та **Ansible**
* **VirtualBox**
* Файл офіційного образу `ubuntu-jammy.box` розміщено у кореневій директорії проєкту (поруч із файлами Terraform).

Також, за потреби, замініть публічний SSH-ключ у файлі `cloud_init.yml` на власний для автоматичного доступу до користувача `ansible`.

---

## Інструкція із запуску

### Етап 1. Provisioning (Terraform)

1. Перейдіть до кореневої директорії проєкту:
   ```bash
   cd terraform
   ```
2. Ініціалізуйте Terraform:
   ```bash
   terraform init
   ```
3. Запустіть розгортання інфраструктури:
   ```bash
   terraform apply
   ```

### Етап 2. Configuration Management (Ansible)

1. Перейдіть до директорії Ansible:
   ```bash
   cd ansible
   ```
2. Перевірте з'єднання з вузлами:
   ```bash
   ansible all -i inventory.ini -m ping
   ```
3. Запустіть налаштування серверів:
   ```bash
   ansible-playbook -i inventory.ini playbook.yml
   ```

## Перевірка працездатності 
Після успішного запуску Terraform та Ansible, систему можна перевірити через HTTP-запити до вузла `worker-vm` (`192.168.56.103`):

* **Головна сторінка веб-застосунку:** `curl http://192.168.56.103/`
* **Health Check 1 (Статус застосунку):** `curl http://192.168.56.103/health/alive` 
* **Health Check 2 (Статус підключення до БД):** `curl http://192.168.56.103/health/ready`
